## Pipeline functions

### `createQueryAgent(config)`

Create a query agent that answers questions against stored documents with citation-backed provenance.
It also supports per-query multimodal attachments such as photos, PDFs, and raw text snippets.

| Parameter | Type | Description |
|-----------|------|-------------|
| `config` | `QueryConfig` | Pipeline configuration (see below) |

**Returns:** `{ query(input: QueryInput) => Promise<QueryOutput> }`

```typescript
import { createQueryAgent } from "@claritylabs/cl-sdk";

const agent = createQueryAgent({
  generateText,
  generateObject,
  documentStore,
  memoryStore,
  concurrency: 3,
  maxVerifyRounds: 1,
  retrievalLimit: 10,
});
```

---

### `agent.query(input)`

Run the full query pipeline: interpret attachments, classify, plan retrieval, retrieve if needed, reason, verify, respond.

| Parameter | Type | Description |
|-----------|------|-------------|
| `input.question` | `string` | The user's question |
| `input.conversationId` | `string?` | Conversation ID for history context |
| `input.context` | `AgentContext?` | Platform and user context |
| `input.attachments` | `QueryAttachment[]?` | Optional inbound files or text context to interpret and combine with stored evidence |

**Returns:** `Promise<QueryOutput>`

```typescript
const result = await agent.query({
  question: "What is the deductible on our GL policy?",
  conversationId: "conv-123",
});

console.log(result.answer);      // Natural language answer with [1], [2] citations
console.log(result.citations);   // Citation[] with source references
console.log(result.confidence);  // 0-1 confidence score
console.log(result.intent);      // "policy_question"
console.log(result.followUp);    // Optional follow-up suggestion
console.log(result.tokenUsage);  // { inputTokens, outputTokens }
console.log(result.reviewReport); // Deterministic grounding/citation quality report
```

Retrieval is bounded by the classifier and planner. Simple general questions and attachment-only questions can skip document/chunk lookup entirely, while document-backed questions still retrieve evidence and verify grounding before responding.

### Multimodal example

```typescript
const result = await agent.query({
  question: "What else do we need to know about this, and does it match the stored policy?",
  conversationId: "conv-123",
  attachments: [
    {
      kind: "image",
      name: "wall-damage.jpg",
      mimeType: "image/jpeg",
      base64: damagePhotoBase64,
      description: "Photo the tenant texted in",
    },
    {
      kind: "pdf",
      name: "coi.pdf",
      mimeType: "application/pdf",
      base64: coiPdfBase64,
    },
    {
      kind: "text",
      name: "email-context",
      text: "Carrier asked whether the certificate holder needs waiver of subrogation.",
    },
  ],
});
```

## Types

### `QueryConfig`

```typescript
interface QueryConfig {
  generateText: GenerateText;
  generateObject: GenerateObject;
  documentStore: DocumentStore;
  memoryStore: MemoryStore;
  concurrency?: number;           // default: 3
  maxVerifyRounds?: number;       // default: 1
  retrievalLimit?: number;        // default: 10
  onTokenUsage?: (usage: TokenUsage) => void;
  onProgress?: (message: string) => void;
  log?: LogFn;
  providerOptions?: Record<string, unknown>;
  qualityGate?: "off" | "warn" | "error";
}
```

### `QueryInput`

```typescript
interface QueryInput {
  question: string;
  conversationId?: string;
  context?: AgentContext;
  attachments?: QueryAttachment[];
}
```

### `QueryAttachment`

```typescript
interface QueryAttachment {
  id?: string;
  kind: "image" | "pdf" | "text";
  name?: string;
  mimeType?: string;
  base64?: string;      // image/pdf content
  text?: string;        // text attachment content
  description?: string; // optional caller hint
}
```

### `QueryOutput`

```typescript
interface QueryOutput {
  answer: string;
  citations: Citation[];
  intent: QueryIntent;
  confidence: number;
  followUp?: string;
  tokenUsage: TokenUsage;
  reviewReport: QueryReviewReport;
}
```

### `Citation`

```typescript
interface Citation {
  index: number;
  chunkId: string;
  documentId: string;
  documentType?: "policy" | "quote";
  field?: string;
  quote: string;
  relevance: number;
}
```

### `QueryIntent`

```typescript
type QueryIntent =
  | "policy_question"
  | "coverage_comparison"
  | "document_search"
  | "claims_inquiry"
  | "general_knowledge";
```

## Query prompt functions

These are exported for advanced use (custom pipelines, testing):

### `buildQueryClassifyPrompt(question, conversationContext?, attachmentContext?)`

Build the classification prompt for intent detection and sub-question decomposition.

### `buildInterpretAttachmentPrompt(question, attachment)`

Build the attachment interpretation prompt used for inbound image/PDF/text context.

### `buildReasonPrompt(subQuestion, intent, evidence)`

Build an intent-specific reasoning prompt with evidence context.

### `buildVerifyPrompt(originalQuestion, subAnswersJson, evidenceJson)`

Build the verification prompt for grounding, consistency, and completeness checks.

### `buildRespondPrompt(originalQuestion, subAnswersJson, platform?)`

Build the final response formatting prompt with platform-specific formatting guidance.

## Provider callback requirements for attachments

The query agent can pass multimodal payloads through `providerOptions` when attachments are present:

- `providerOptions.attachments` for the raw attachment list
- `providerOptions.pdfBase64` for PDF attachments
- `providerOptions.images` for image attachments

Your `generateObject` callback must translate those fields into actual provider-specific file or image parts. If it does not, attachment-aware queries will silently degrade into text-only behavior.
