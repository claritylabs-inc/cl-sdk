All notable changes to `@claritylabs/cl-sdk` are documented here.

## Unreleased ‚Äî Extraction Completeness

### Added

- **First-class definitions extraction** ‚Äî new `definitions` extractor and `document.definitions[]` field for defined terms, full definition text, page/form provenance, section references, and source snippets.
- **First-class covered reasons extraction** ‚Äî new `covered_reasons` extractor and `document.coveredReasons[]` field for covered causes of loss, covered losses, inclusions, qualifying reasons, attached conditions/exceptions, and provenance.
- **Dedicated retrieval chunks** ‚Äî new `definition` and `covered_reason` chunk types. Covered-reason conditions are emitted as separate `covered_reason` chunks for precise retrieval.
- **Quality checks** ‚Äî deterministic warnings for mapped definitions/covered-reason pages with empty structured output, missing definition/covered-reason content, missing page provenance, and referential covered-reason text.
- **Bounded workflow planners** ‚Äî query, application processing, referential coverage resolution, and generic workflow helpers can now skip unnecessary subagents/tool chains while preserving targeted retry paths.
- **Focused extractor fallbacks** ‚Äî fallback-capable extractors can route through section extraction when focused output fails or comes back empty.

### Changed

- **Extraction pipeline now has 13 focused extractors** with page-map routing and review follow-up support for `definitions` and `covered_reasons`.
- **Supplementary extraction is now conditional** based on page assignments, form inventory, extraction signals, or review follow-up tasks.
- **Formatter coverage expanded and cost-gated** to definition text plus covered-reason content and conditions while skipping plain prose that does not need cleanup.
- **Query retrieval and application optional agents are now gated** by classifier/planner decisions to reduce cost and improve edge-case routing.
- **Financial promotion expanded** to more premium and total-cost aliases and declaration-derived `taxesAndFees[]`, including common GST/HST/PST/QST-style fields.

## v0.11.0 ‚Äî Declarations Promotion & Summary Generation

### Added

- **Declarations-to-schema promotion pass** ‚Äî new Phase 7 in the extraction pipeline. After assembly, a promotion pass fills top-level typed fields from extracted declarations and coverage arrays. This bridges the gap between detailed extraction (100+ declaration fields, dozens of coverages) and the typed `InsuranceDocument` schema. Runs seven sub-passes with no model calls:
  - **Carrier field mapping** ‚Äî `naicNumber` ‚Üí `carrierNaicNumber`, `admittedStatus` ‚Üí `carrierAdmittedStatus`, `amBestRating` ‚Üí `carrierAmBestRating`; builds `insurer` sub-object
  - **Broker promotion** ‚Äî declarations broker/agent/producer ‚Üí `brokerAgency`, `brokerContactName`, `producer`
  - **Loss payee / mortgage holder promotion** ‚Äî declarations ‚Üí `lossPayees[]`, `mortgageHolders[]`
  - **Location promotion** ‚Äî groups declaration location fields ‚Üí `locations[]`
  - **Limits synthesis** ‚Äî coverage names ‚Üí `limits` schedule (GL, auto, umbrella, WC patterns)
  - **Deductible synthesis** ‚Äî most common deductible ‚Üí `deductibles.perOccurrence`; detects SIR and aggregate
  - **Premium fallback** ‚Äî declarations premium ‚Üí `premium` when extractor missed it
  - New file: `src/extraction/promote.ts`
- **Summary generation** ‚Äî new Phase 8. Generates a 1‚Äì3 sentence AI summary of the assembled document. Appears as `summary` on the final `InsuranceDocument`. Gracefully skipped on failure.
  - New file: `src/prompts/coordinator/summarize.ts`
- **Broker extraction in `carrier_info`** ‚Äî extractor now captures `brokerAgency`, `brokerContactName`, and `brokerLicenseNumber` alongside carrier data
- **Loss payee / mortgage holder extraction in `named_insured`** ‚Äî extractor now captures `lossPayees[]` and `mortgageHolders[]` alongside insured data

### Fixed

- **Field name inconsistency** ‚Äî carrier extractor output used short names (`naicNumber`, `admittedStatus`) that didn't match the `PolicyDocument` schema's prefixed names (`carrierNaicNumber`, `carrierAdmittedStatus`). The promotion pass now maps these correctly.

### Changed

- Pipeline now runs **10 phases** (was 8): Classify ‚Üí Page Map ‚Üí Plan ‚Üí Extract ‚Üí Review ‚Üí Assemble ‚Üí **Promote** ‚Üí **Summarize** ‚Üí Format ‚Üí Chunk

## v0.7.x ‚Äî Schema Compatibility & Resilience

### Added

- **`toStrictSchema()`** ‚Äî auto-transforms Zod schemas before `generateObject` calls, converting `.optional()` to `.nullable()` for OpenAI strict structured output compatibility. Applied automatically in the pipeline.
- **`safeGenerateObject()`** ‚Äî wraps `generateObject` with schema strictification, retry on schema validation errors, and optional fallback values when all retries are exhausted.
- **Pipeline checkpoints** ‚Äî `createPipelineContext()` for save/resume at phase boundaries with `onCheckpointSave` callback.

### Fixed

- **`z.record()` removal** ‚Äî replaced all `z.record()` usages in schemas with `z.array(z.object({ key, value }))` to avoid unsupported `propertyNames` in JSON Schema.
- **Transient error retry** ‚Äî `withRetry()` now retries on grammar compilation timeouts, no-output-generated, overloaded, and 5xx errors (not just rate limits).
- **Graceful degradation** ‚Äî extraction pipeline uses fallback values for classify/plan/review when all retries fail, allowing the pipeline to continue rather than crash.

## v0.5.0

### Added

- **Markdown formatting pass** ‚Äî new Phase 6 in the extraction pipeline. After assembly, a formatting agent cleans up all content-bearing fields (sections, endorsements, exclusions, conditions, summary) to ensure proper markdown rendering. Fixes pipe tables missing separator rows, space-aligned tables, sub-items mixed into tables, and general whitespace issues.
  - New files: `src/extraction/formatter.ts`, `src/prompts/coordinator/format.ts`
  - Pipeline now runs 7 phases: Classify ‚Üí Plan ‚Üí Extract ‚Üí Review ‚Üí Assemble ‚Üí Format ‚Üí Chunk
  - Token usage from formatting is tracked and included in the total

## v0.2.0 ‚Äî Upcoming

- **BREAKING:** Removed `SONNET_MODEL` and `HAIKU_MODEL` constants

## v0.1.0 ‚Äî 2026-04-11

Version reset. All prior versions (1.x through 6.x) are no longer relevant and should not be used. The SDK will remain below 1.0.0 until official launch.

### Current Features

- Document classification (`classifyDocumentType`)
- PDF extraction pipeline (`extractFromPdf`, `extractQuoteFromPdf`)
- Provider-agnostic model configuration via Vercel AI SDK `LanguageModel` instances
- Composable agent prompt system (`buildAgentSystemPrompt`)
- Platform and intent model for multi-channel agents
- PDF operations (AcroForm fill, text overlay, page splitting)
- Comprehensive insurance type system (42 policy types, declarations, coverages)
- Rate-limit retry with exponential backoff
- Parallel chunk extraction with configurable concurrency
- Token usage tracking via `onTokenUsage` callback
- Context key mapping for policy-to-application auto-fill
- Native PDF file support with `convertPdfToImages` fallback
