## v0.11.0 — Declarations Promotion & Summary Generation

### Added

- **Declarations-to-schema promotion pass** — new Phase 7 in the extraction pipeline. After assembly, a promotion pass fills top-level typed fields from extracted declarations and coverage arrays. This bridges the gap between detailed extraction (100+ declaration fields, dozens of coverages) and the typed `InsuranceDocument` schema. Runs seven sub-passes with no model calls:
  - **Carrier field mapping** — `naicNumber` → `carrierNaicNumber`, `admittedStatus` → `carrierAdmittedStatus`, `amBestRating` → `carrierAmBestRating`; builds `insurer` sub-object
  - **Broker promotion** — declarations broker/agent/producer → `brokerAgency`, `brokerContactName`, `producer`
  - **Loss payee / mortgage holder promotion** — declarations → `lossPayees[]`, `mortgageHolders[]`
  - **Location promotion** — groups declaration location fields → `locations[]`
  - **Limits synthesis** — coverage names → `limits` schedule (GL, auto, umbrella, WC patterns)
  - **Deductible synthesis** — most common deductible → `deductibles.perOccurrence`; detects SIR and aggregate
  - **Premium fallback** — declarations premium → `premium` when extractor missed it
  - New file: `src/extraction/promote.ts`
- **Summary generation** — new Phase 8. Generates a 1–3 sentence AI summary of the assembled document. Appears as `summary` on the final `InsuranceDocument`. Gracefully skipped on failure.
  - New file: `src/prompts/coordinator/summarize.ts`
- **Broker extraction in `carrier_info`** — extractor now captures `brokerAgency`, `brokerContactName`, and `brokerLicenseNumber` alongside carrier data
- **Loss payee / mortgage holder extraction in `named_insured`** — extractor now captures `lossPayees[]` and `mortgageHolders[]` alongside insured data

### Fixed

- **Field name inconsistency** — carrier extractor output used short names (`naicNumber`, `admittedStatus`) that didn't match the `PolicyDocument` schema's prefixed names (`carrierNaicNumber`, `carrierAdmittedStatus`). The promotion pass now maps these correctly.

### Changed

- Pipeline now runs **10 phases** (was 8): Classify → Page Map → Plan → Extract → Review → Assemble → **Promote** → **Summarize** → Format → Chunk

## v0.7.x — Schema Compatibility & Resilience

### Added

- **`toStrictSchema()`** — auto-transforms Zod schemas before `generateObject` calls, converting `.optional()` to `.nullable()` for OpenAI strict structured output compatibility. Applied automatically in the pipeline.
- **`safeGenerateObject()`** — wraps `generateObject` with schema strictification, retry on schema validation errors, and optional fallback values when all retries are exhausted.
- **Pipeline checkpoints** — `createPipelineContext()` for save/resume at phase boundaries with `onCheckpointSave` callback.

### Fixed

- **`z.record()` removal** — replaced all `z.record()` usages in schemas with `z.array(z.object({ key, value }))` to avoid unsupported `propertyNames` in JSON Schema.
- **Transient error retry** — `withRetry()` now retries on grammar compilation timeouts, no-output-generated, overloaded, and 5xx errors (not just rate limits).
- **Graceful degradation** — extraction pipeline uses fallback values for classify/plan/review when all retries fail, allowing the pipeline to continue rather than crash.

## v0.5.0

### Added

- **Markdown formatting pass** — new Phase 6 in the extraction pipeline. After assembly, a formatting agent cleans up all content-bearing fields (sections, endorsements, exclusions, conditions, summary) to ensure proper markdown rendering. Fixes pipe tables missing separator rows, space-aligned tables, sub-items mixed into tables, and general whitespace issues.
  - New files: `src/extraction/formatter.ts`, `src/prompts/coordinator/format.ts`
  - Pipeline now runs 7 phases: Classify → Plan → Extract → Review → Assemble → Format → Chunk
  - Token usage from formatting is tracked and included in the total

## v0.2.0 — Upcoming

### Changes

- **BREAKING:** `models` parameter is now required for all extraction functions (`ExtractOptions`, `ClassifyOptions`, `ExtractSectionsOptions`)
- **BREAKING:** Removed `createDefaultModelConfig()` — consumers must provide their own models
- **BREAKING:** Removed `SONNET_MODEL` and `HAIKU_MODEL` constants
- Added `pdfContentFormat` and `convertPdfToImages` options for PDF input format control
- Added `tokenLimits` option for per-role token limit overrides
- `MODEL_TOKEN_LIMITS` is now deprecated — use `DEFAULT_TOKEN_LIMITS`
- Fixed default metadata token limit to 16,384 (was incorrectly documented as 4,096)

## v0.1.0 — 2026-04-11

Version reset. All prior versions (1.x through 6.x) are no longer relevant and should not be used. The SDK will remain below 1.0.0 until official launch.

### Current Features

- Document classification (`classifyDocumentType`)
- PDF extraction pipeline (`extractFromPdf`, `extractQuoteFromPdf`)
- Provider-agnostic model configuration via Vercel AI SDK `LanguageModel` instances
- Composable agent prompt system (`buildAgentSystemPrompt`)
- Platform and intent model for multi-channel agents
- PDF operations (AcroForm fill, text overlay, page splitting)
- Comprehensive insurance type system (42 policy types, declarations, coverages)
- Rate-limit retry with exponential backoff
- Parallel chunk extraction with configurable concurrency
- Token usage tracking via `onTokenUsage` callback
- Context key mapping for policy-to-application auto-fill
- Native PDF file support with `convertPdfToImages` fallback
